Requirements

  • linux based server
  • A web server is required to serve the requested pages to the client browser.
  • PHP is required to run the software.

Linux Server

CentOS shall be used as the operating system of choice for this project. It comes pre-packaged with Apache and the necessary tools for managing the wiki.

Web Server

The webserver will be run by the CSE data center; using whatever software is provided there.

As of 2013-10-14 (see the testlog) Apache-1.x will be the webserver software to be used to serve the wiki pages. See below for details on isntalling and configuring Apache.

PHP

PHP will be required to run the web applications written for DokuWiki. The most updated version is prefered.

Installing Apache and PHP

All software should be installed using the package-manager of the server os' choice (for CentOS it is yum). CentOs should come pre-installed with Apache.

If PHP for some reason is not installed please refer to the CentOS website for which package to install and how to configure it.

Configuring CSEWiki

The Dokuwiki software can be configured using the web-interface or by hand editing the text files associated with whatever your configuration desires are. Please see the dokuwiki configuration page before attempting any configuration.

Templates (Skins)

The GUI appearance of a dokuwiki template (in mediawiki terms it's a skin) can be customized easily through editing a series of css and php files. The CSEWiki template has been created using the dokuwiki starter template. See the dokuwiki templates page for details on how to configure the template.

Page Management

Pages are organized into namespaces. Each page has a "long" name with the name of the page being preceded by parent namespace names followed by a collon. For example: 

    namespace1:namespace2:page_name.

The physical location of a page is: data/pages/ . Each page is a textfile so managing them is extremely easy. Below is an example location of a page. 

    data/pages/namespace1/namespace2/page_name.txt

Note: the namespaces are actually directories.

Below are the default namespaces to be created for the csewiki (these may change with time).

Topic(Namespace)

Labs(lab)

lab related topics (printing, computer lab access, lab access, etc..) should be located atleast under the lab namespace.

Tutorial(tutorial)

programing related tutorials/examples should be located under the tutorial namespace.

Faculty(staff)

faculty/staff related how-to pages are to be under the staff namespace.

Misc()

any topic that doesn't fit the above namespace should be namespaceless. If there becomes a substancial number of articles with a smiliar topic then a new namespace should be created.

User Management

User permissions are managed through dokuwiki's ACL system. Their is a web-based front end for this configuration however you need to be logged in as root to access it. See the dokuwiki acl page for details.

Superuser

There is to be only one super user for the entire wiki. The password for the super user should never be shared, and only one physical person should know the password. The superuser is called root (after the Linux superuser). This user will have 100% control over the wiki and will be only user to manage the ACL, users, and wiki configuration.

Any kind of new user that is created can only be created by the root user.

Admins

Those given the admin privileges will have the ability to create and delete pages (user full rights). They will have no management privileges (they won't be able to access the ACL, wiki config, or user management). Admins are to administrate the content of the wiki, not the configuration of the wiki (that is left to the superuser).

Currently only admins will have creating/editing privileges for the wiki.

New Users

Currently there is no need for regular users. By default regular users will only be able to read wiki pages (which you do not have to be logged-in to do).
