2013-10-16

Change of Plan

After 2 months of attempting to configure MediaWiki software I began to realize the difficulty in performing basic management tasks. For instance creating a custom wiki skin involves writing thousands of PHP code which becomes very inefficient to manage or create. Also configuring user permissions requires the mantainer of a MediaWiki project write many lines of code for repetative tasks. While MediaWiki is very powerful it requires an entire team of developers to create/maintain which calls for a more expensive (cost and resources) project. Another issue is all data for MediaWiki is stored in a SQL server which adds another level of complexity to the project itself.

After installing DokuWiki on my personal apache web server (DokuWiki does not require a database server) I found creating/managing the wiki is simpler than MediaWiki. Creating a new look and feel for the wiki took only a couple hours and the learning curve was a breeze. If this project is to remain as a low budget and easy to maitain project than I will be changing the wiki engine software to DokuWiki.

Note: Due to this change install.txt has now been moved to installmwiki.txt for record keeping (later it will be moved as git has a great system for managing files). installdwiki.txt has been created to explain the installation and configuration process for DokuWiki.
